样本级| 97.06%| 64.71%| 77.65%| Recall 偏低

##  二、推荐方案（按优先级排序）

### 🔥 方案 1: 引入 PyOD 异常检测算法集成（最高优先级）

**来源** : PyOD 库 (JMLR 2019, NeurIPS 2022)  
**GitHub** : [https://github.com/yzhao062/pyod](https://github.com/yzhao062/pyod) (26M+ 下载)  
**论文** : [https://arxiv.org/abs/2412.12154](https://arxiv.org/abs/2412.12154)

#### 推荐理由

  * 包含 50+ 种异常检测算法，从经典 LOF 到最新 ECOD、DIF


  * 统一 API，易于集成到现有 pipeline


  * 支持 LLM-powered model selection（自动选择最佳算法）


  * 可解释性强，输出 anomaly score 而非仅 binary label


#### 具体实施
```text
    from pyod.models.ecod import ECOD
    from pyod.models.iforest import IForest
    from pyod.models.lof import LOF
    
    # 为每个元器件构建特征向量
    # 特征包括：rule_score, support_view_count, symmetry_group_size, 
    #          peer_count, net_size, value_deviation 等
    
    # 集成多个检测器
    detectors = {
        'ecod': ECOD(),
        'iforest': IForest(n_estimators=200),
        'lof': LOF(n_neighbors=20)
    }
    
    # 对每个候选元器件进行异常评分
    scores = {}
    for name, clf in detectors.items():
        clf.fit(X_train)
        scores[name] = clf.decision_function(X_candidates)
    
    # 融合多个检测器的分数（加权平均或 max）
    final_scores = np.mean([scores[k] for k in scores], axis=0)
```

#### 预期收益

  * **Recall 提升 15-25%** (基于 ADBench 57 个数据集平均结果)


  * 对局部异常（local anomalies）特别有效


  * 几乎不增加推理时间（ECOD 无需训练）


#### 参考资源

  * ADBench Benchmark: [https://github.com/Minqi824/ADBench](https://github.com/Minqi824/ADBench) (NeurIPS 2022)


  * 57 个数据集、30 种算法的系统性对比


  * 关键发现：无监督方法中 ECOD、IForest 表现最稳健

### 🔥 方案 2: 引入图神经网络异常检测（PyGOD）

**来源** : PyGOD 库 (JMLR 2024, NeurIPS 2022 BOND)  
**GitHub** : [https://github.com/pygod-team/pygod](https://github.com/pygod-team/pygod)  
**论文** : [https://jmlr.org/papers/v25/23-0963.html](https://jmlr.org/papers/v25/23-0963.html)

#### 推荐理由

  * 专门针对**图结构数据** 的异常检测


  * 电路原理图天然是图结构（元器件=节点，连接=边）


  * 10+ 种图异常检测算法（DOMINANT、GAE、OCGNN 等）


  * 能捕捉拓扑结构异常（对称性、邻域模式）


#### 具体实施
```text
    from pygod.detector import DOMINANT, OCGNN
    import torch
    from torch_geometric.data import Data
    
    # 构建电路图
    # node_features: [num_components, feature_dim]
    # edge_index: [2, num_connections]
    # edge_features: [num_connections, edge_feature_dim]
    
    graph_data = Data(
        x=node_features,
        edge_index=edge_index,
        edge_attr=edge_features
    )
    
    # 使用 DOMINANT (GNN + Autoencoder)
    model = DOMINANT(num_layers=4, epoch=50, lr=0.001)
    model.fit(graph_data)
    
    # 获取节点级异常分数
    node_scores = model.decision_score_
    
    # 融合到现有决策系统
    # final_score = alpha * rule_score + beta * gnn_score
```

#### 预期收益

  * **Recall 提升 20-30%** (针对拓扑相关错误)


  * 特别擅长检测：对称拓扑异常、邻域模式异常


  * 与现有规则系统互补（规则=显式知识，GNN=隐式模式）


#### 参考资源

  * BOND Benchmark: [https://proceedings.neurips.cc/paper_files/paper/2022/hash/acc1ec4a9c780006c9aafd595104816b-Abstract-Datasets_and_Benchmarks.html](https://proceedings.neurips.cc/paper_files/paper/2022/hash/acc1ec4a9c780006c9aafd595104816b-Abstract-Datasets_and_Benchmarks.html)

### 🔥 方案 4: 引入外部元器件知识库

#### 4.1 E-series 标准值扩展

**来源** : IEC 60063 标准  
**维基百科** : [https://en.wikipedia.org/wiki/E_series_of_preferred_numbers](https://en.wikipedia.org/wiki/E_series_of_preferred_numbers)

##### 实施
```text
    # 扩展现有 E-series 检测
    E_SERIES = {
        'E6': [1.0, 1.5, 2.2, 3.3, 4.7, 6.8],
        'E12': [1.0, 1.2, 1.5, 1.8, 2.2, 2.7, 3.3, 3.9, 4.7, 5.6, 6.8, 8.2],
        'E24': [1.0, 1.1, 1.2, 1.3, 1.5, 1.6, 1.8, 2.0, 2.2, 2.4, 2.7, 3.0, 
                3.3, 3.6, 3.9, 4.3, 4.7, 5.1, 5.6, 6.2, 6.8, 7.5, 8.2, 9.1],
        'E48': [...],  # 48 个值
        'E96': [...],  # 96 个值
        'E192': [...]  # 192 个值
    }
    
    def check_e_series_compliance(value, tolerance='5%'):
        """
        检查数值是否符合 E-series 标准
        返回：(is_compliant, closest_standard_value, series_name)
        """
        # 根据 tolerance 选择合适的 series
        if tolerance == '5%':
            series = E_SERIES['E24']
        elif tolerance == '1%':
            series = E_SERIES['E96']
        # ...
        
        # 检查 value 是否在标准值的容差范围内
        for std_val in series:
            if abs(value - std_val) / std_val <= tolerance:
                return True, std_val, series_name
        
        return False, None, None
```

```text
        """
        根据元器件类型和数值范围进行验证

            return {'valid': True, 'reason': 'unknown_type'}
        
        db_entry = component_db[component_type]

        # 检查是否在常见值附近（考虑数量级）

        for std_val in common_values:
            if is_close(normalized_value, std_val, tolerance=0.1):
                return {'valid': True, 'reason': 'common_value'}
        
        # 如果不在常见值列表中，标记为可疑
```

### 🔥 方案 5: 集成学习 + 校准

**来源** : ADBench 关键发现 (NeurIPS 2022)  
**论文** : [https://arxiv.org/abs/2206.09426](https://arxiv.org/abs/2206.09426)

#### 推荐理由

  * ADBench 分析 57 个数据集后发现：**单一算法无法在所有场景最优**


  * 集成方法（ensemble）通常比单一方法稳健 10-15%


  * 校准（calibration）可显著提高 threshold 选择的准确性


#### 具体实施

##### 5.1 多模型集成
```text
    from sklearn.ensemble import VotingClassifier
    from pyod.combination import average, maximization
    
    # 现有系统已有：rule_score, LLM_candidate, LLM_verifier
    # 新增：PyOD detectors, PyGOD detectors
    
    all_scores = {
        'rule_based': rule_scores,
        'llm_candidate': candidate_scores,
        'llm_verifier': verifier_scores,
        'pyod_ecod': ecod_scores,
        'pyod_iforest': iforest_scores,
        'pygod_dominant': gnn_scores
    }
    
    # 策略 1: 简单平均
    ensemble_score_avg = np.mean([all_scores[k] for k in all_scores], axis=0)
    
    # 策略 2: 加权平均（根据历史表现分配权重）
    weights = {
        'rule_based': 0.25,
        'llm_verifier': 0.25,
        'pyod_ecod': 0.15,
        'pyod_iforest': 0.15,
        'pygod_dominant': 0.20
    }
    ensemble_score_weighted = sum(all_scores[k] * weights[k] for k in weights)
    
    # 策略 3: Max 融合（保守策略，提高 Recall）
    ensemble_score_max = np.max([all_scores[k] for k in all_scores], axis=0)
```

##### 5.2 阈值校准
```text
    from sklearn.calibration import CalibratedClassifierCV
    import numpy as np
    
    # 使用验证集校准分数到概率
    def calibrate_scores(scores, labels, method='isotonic'):
        """
        将 anomaly scores 校准为概率
        method: 'sigmoid' (Platt scaling) or 'isotonic'
        """
        calibrator = CalibratedClassifierCV(method=method, cv=5)
        # 需要转换为 classification 格式
        X = scores.reshape(-1, 1)
        y = labels
        calibrator.fit(X, y)
        return calibrator.predict_proba(X)[:, 1]
```

##### 5.3 动态阈值选择
```text
    def select_threshold_dynamically(scores, target_recall=0.8):
        """
        根据目标 Recall 动态选择阈值
        """
        # 在验证集上搜索最佳阈值
        thresholds = np.percentile(scores, np.arange(0, 100, 1))
        best_threshold = None
        best_f1 = 0
        
        for thresh in thresholds:
            preds = (scores >= thresh).astype(int)
            precision, recall, f1 = compute_metrics(preds, y_val)
            
            if recall >= target_recall and f1 > best_f1:
                best_f1 = f1
                best_threshold = thresh
        
        return best_threshold
```

## 三、实施路线图

### 阶段 1: 快速见效（1-2 周）

  3. **多模型集成 + 加权融合**


**预期** : Recall 再提升 10-15%，Precision 提升 5-10%

### 阶段 3: 长期改进（1-2 月）

  2. **阈值校准 + 动态选择** \- 需要验证集


  3. **Targeted Verifier 优化** \- 对 rule_only 候选单独处理


**预期** : F1 达到 0.55-0.65（当前 0.37）
