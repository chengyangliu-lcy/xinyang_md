# 按目标服务器路径适配的增量灌库迁移方案

## Summary

本服务器负责做完整增量：新 PDF → MinerU 解析 → Markdown/content_list → 向量化入 Milvus → 重建 BM25。目标服务器无 GPU，只接收已经处理好的数据和库文件。

目标服务器路径按你提供的为准：

类型| 目标服务器路径  
---|---  
原始 PDF 根目录| `/home/xinyang/material/Content/UploadFiles`  
Excel| `/home/xinyang/material/data/work/物料规格书对照表1.xlsx`  
Milvus 卷| `/home/xinyang/material/milvus/volumes`  
模型| `/home/xinyang/material/models/Qwen3-Embedding-4B`  
Markdown/content_list| `/home/xinyang/material/xinyang_sch/markdownfiles`  
  
目标机不需要重新跑 MinerU，也不需要重新向量化入库。

## 本服务器执行

  1. 把新增 PDF 放入本服务器现有 PDF 根目录，保持和目标机同样的相对结构。例如目标机最终应是：


```text
    /home/xinyang/material/Content/UploadFiles/201911/xx/xxx.pdf
```

本服务器可对应放在：
```text
    /data/xinyang_data/Content/UploadFiles/201911/xx/xxx.pdf
```

  2. 更新本服务器 Excel。新增 PDF 必须在 `物料规格书对照表1.xlsx` 里有行，关键列：


```text
    机种, 物料编码, 物料名称, 规格型号, 类别, 文档地址
```

`文档地址` 推荐写：
```text
    /Content/UploadFiles/201911/xx/xxx.pdf
```

  3. 跑 MinerU 解析并确认产物。增量 PDF 解析后，本服务器必须生成：


```text
    /data/xinyang_sch/data/markdown_files///auto/_content_list.json
    /data/xinyang_sch/data/markdown_files///auto/.md
```

  4. 更新映射 JSON。本服务器映射文件：


```text
    /data/xinyang/milvus_new/pdf_markdown_mapping_merged.json
```

新增项应形如：
```text
    {
      "tmpxxxx": "/data/xinyang_data/Content/UploadFiles/201911/xx/xxx.pdf"
    }
```

  5. 本服务器执行增量灌库。


```text
    export MATERIAL_COLLECTION_NAME=material_documents_qwen3_full
    export BGE_MODEL_PATH=/data/music/models/Qwen3-Embedding-4B
    export MARKDOWN_DIR=/data/xinyang_sch/data/markdown_files
    export EXCEL_FILE=/data/xinyang/物料规格书对照表1.xlsx
    export PDF_MAPPING_FILE=/data/xinyang/milvus_new/pdf_markdown_mapping_merged.json
    export PDF_MAPPING_FILE_BACKUP=/data/xinyang/milvus_new/pdf_markdown_mapping_merged.json
    
    cd /home/music/lcy/material-rag/02_ingestion
    python import_content_list_v2.py \
      --collection material_documents_qwen3_full \
      --gpu 0 \
      --no-create \
      --yes
```

  6. 本服务器重建 BM25。


```text
    cd /home/music/lcy/material-rag/03_retrieval_api
    python bm25_index.py --rebuild \
      --collection material_documents_qwen3_full \
      --output bm25_cache_material_documents_qwen3_full.pkl
```

  7. 本服务器验证新增文档能检索到，PDF 链接可打开。


## 同步到目标服务器

设目标机变量：
```text
    TARGET_USER=xinyang
    TARGET_HOST=<目标服务器IP>
    TARGET_CODE_DIR=<目标服务器当前代码目录>
```

目标机先停 API 和 Milvus：
```text
    ssh ${TARGET_USER}@${TARGET_HOST}
    # 在目标机执行
    cd ${TARGET_CODE_DIR}/05_milvus_deploy
    export DOCKER_VOLUME_DIRECTORY=/home/xinyang/material/milvus
    docker compose down
    # 停 API 进程，按目标机现有方式执行
```

本服务器同步数据：
```text
    # 1. PDF：同步到目标机 PDF 根目录
    rsync -aH /data/xinyang_data/Content/UploadFiles/ \
      ${TARGET_USER}@${TARGET_HOST}:/home/xinyang/material/Content/UploadFiles/
    
    # 2. Markdown/content_list：注意目标机目录叫 markdownfiles
    rsync -aH /data/xinyang_sch/data/markdown_files/ \
      ${TARGET_USER}@${TARGET_HOST}:/home/xinyang/material/xinyang_sch/markdownfiles/
    
    # 3. Excel
    rsync -aH /data/xinyang/物料规格书对照表1.xlsx \
      ${TARGET_USER}@${TARGET_HOST}:/home/xinyang/material/data/work/物料规格书对照表1.xlsx
    
    # 4. Milvus 卷
    rsync -aH --delete /data/xinyang_milvus/volumes/ \
      ${TARGET_USER}@${TARGET_HOST}:/home/xinyang/material/milvus/volumes/
    
    # 5. BM25 缓存
    rsync -aH /home/music/lcy/material-rag/03_retrieval_api/bm25_cache_material_documents_qwen3_full.pkl \
      ${TARGET_USER}@${TARGET_HOST}:${TARGET_CODE_DIR}/03_retrieval_api/bm25_cache_material_documents_qwen3_full.pkl
```

映射 JSON 需要特别处理：目标机 PDF 路径和本服务器不同，建议在同步前生成目标机专用映射文件。

在本服务器生成一份目标机映射副本：
```text
    python - <<'PY'
    import json
    
    src = "/data/xinyang/milvus_new/pdf_markdown_mapping_merged.json"
    dst = "/tmp/pdf_markdown_mapping_merged.target.json"
    
    with open(src, "r", encoding="utf-8") as f:
        data = json.load(f)
    
    new = {}
    for k, v in data.items():
        if isinstance(v, str):
            v = v.replace(
                "/data/xinyang_data/Content/UploadFiles",
                "/home/xinyang/material/Content/UploadFiles",
            )
            v = v.replace(
                "/data/xinyang/export_for_teacher/Content/UploadFiles",
                "/home/xinyang/material/Content/UploadFiles",
            )
        new[k] = v
    
    with open(dst, "w", encoding="utf-8") as f:
        json.dump(new, f, ensure_ascii=False)
    
    print(dst, len(new))
    PY
```

同步目标机专用映射：
```text
    rsync -aH /tmp/pdf_markdown_mapping_merged.target.json \
      ${TARGET_USER}@${TARGET_HOST}:/home/xinyang/material/data/work/pdf_markdown_mapping_merged.json
```

## 目标服务器执行

目标机设置运行环境变量。重点是路径必须换成目标机路径：
```text
    export MATERIAL_COLLECTION_NAME=material_documents_qwen3_full
    export BGE_MODEL_PATH=/home/xinyang/material/models/Qwen3-Embedding-4B
    export PDF_PATH_PREFIX=/home/xinyang/material/Content/UploadFiles
    export MARKDOWN_PATH_PREFIX=/home/xinyang/material/xinyang_sch/markdownfiles
    export MAPPING_FILE_PATH=/home/xinyang/material/data/work/pdf_markdown_mapping_merged.json
    export EXCEL_FILE=/home/xinyang/material/data/work/物料规格书对照表1.xlsx
    export MARKDOWN_DIR=/home/xinyang/material/xinyang_sch/markdownfiles
```

启动 Milvus：
```text
    cd ${TARGET_CODE_DIR}/05_milvus_deploy
    export DOCKER_VOLUME_DIRECTORY=/home/xinyang/material/milvus
    docker compose up -d
```

启动 API：
```text
    cd ${TARGET_CODE_DIR}/03_retrieval_api
    export CONDA_ENV=milvus_vectorization
    export API_HOST=0.0.0.0
    export API_PORT=5010
    nohup bash start_api.sh > api_server.log 2>&1 &
    echo $! > api_server.pid
```

验证：
```text
    curl http://localhost:5010/health
    curl http://localhost:5010/stats
    
    curl -X POST http://localhost:5010/search/dify \
      -H "Content-Type: application/json" \
      -d '{"mode":"text","content":"<新增物料编码或关键词>","top_k":3}'
```

验收标准：

  * `/stats` 的 collection 数量和本服务器一致。
  * 新增文档关键词能召回。
  * 返回结果中的 PDF URL 能打开。
  * API 日志没有 `mapping file not found`、`bm25 cache not found`、`collection not found`。


## Rollback

同步前在目标机备份三类核心数据：
```text
    mv /home/xinyang/material/milvus/volumes \
      /home/xinyang/material/milvus/volumes_backup_$(date +%Y%m%d_%H%M)
    
    cp /home/xinyang/material/data/work/物料规格书对照表1.xlsx \
      /home/xinyang/material/data/work/物料规格书对照表1.xlsx.backup_$(date +%Y%m%d_%H%M)
    
    cp /home/xinyang/material/data/work/pdf_markdown_mapping_merged.json \
      /home/xinyang/material/data/work/pdf_markdown_mapping_merged.json.backup_$(date +%Y%m%d_%H%M)
```

如目标机启动后异常，停 API 和 Milvus，把 `volumes_backup_*` 改回 `volumes`，再启动即可。

## Assumptions

  * 目标机代码已经存在且无需同步代码。
  * 目标机目录 `markdownfiles` 对应本服务器的 `markdown_files`，只改路径名，不改内部 `<tmp>/<tmp>/auto/...` 结构。
  * 目标机模型已存在，所以不重复同步 `/home/xinyang/material/models/Qwen3-Embedding-4B`。
  * 迁移 Milvus 卷必须在目标机 Milvus 停止后执行；推荐本服务器 Milvus 也停止后同步，避免卷文件处于写入状态。
