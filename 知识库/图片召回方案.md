## 方案一：文本检索 + 页面截图返回（最快上线）

**特点**：复用现有文本检索，根据命中结果返回对应页面截图。

### 架构

```
文档入库阶段：
PDF → 文本切片 + embedding → Milvus 入库
     → 每页渲染 PNG → 保存 page_no → page_image_url 映射

查询阶段：
OpenWebUI Tool
  ↓
/search/dify 文本检索
  ↓
返回 doc_address + page_no
  ↓
后端补充 page_image_url
  ↓
OpenWebUI 展示文本 + 页面截图
```

### 目录结构

```
/data/xinyang_knowledge/
  pdf/
    doc123.pdf
  pdf_pages/
    doc123/
      page_0001.png
      page_0002.png
  page_manifest/
    doc123.json
```

`doc123.json`:

```json
{
  "doc_id": "doc123",
  "doc_address": "/pdf/doc123.pdf",
  "pages": {
    "1": "/pdf_pages/doc123/page_0001.png",
    "2": "/pdf_pages/doc123/page_0002.png"
  }
}
```

### 返回格式

```json
{
  "results": [
    {
      "result_type": "text_with_page_image",
      "score": 0.83,
      "doc_name": "电芯装配SOP.pdf",
      "doc_address": "/pdf/doc123.pdf",
      "url": "http://10.1.120.36:5010/pdf/doc123.pdf",
      "page_no": 7,
      "content": "电芯装配流程包括上料、扫码、焊接...",
      "page_image_url": "http://10.1.120.36:5010/pdf_pages/doc123/page_0007.png",
      "neighbor_page_image_urls": [
        "http://10.1.120.36:5010/pdf_pages/doc123/page_0006.png",
        "http://10.1.120.36:5010/pdf_pages/doc123/page_0007.png",
        "http://10.1.120.36:5010/pdf_pages/doc123/page_0008.png"
      ]
    }
  ]
}
```

### 优点

- 改动最小，最快上线
- 不需要升级 Milvus
- 不需要图像模型
- 用户看得到图

### 缺点

- 不是真正的视觉检索
- 依赖文本检索准确性
- 需要入库时保留 page_no

### 关键问题：文本检索没有 page_no 怎么办

| 情况 | 解决办法 |
|------|----------|
| chunk 已有 page_no | 直接返回同页截图 |
| 只有 doc_address | 返回前 N 页预览 |
| 没有页码信息 | 入库时补齐 chunk + page_no 映射 |

### 适用场景

- SOP、流程图、规格书页面查找
- "查电芯装配流程图" → 返回对应 SOP 页面截图

## 方案二：OCR/Caption 转文本 + 文本检索

**特点**：把图片转成文字描述（caption），写入原有文本知识库，检索 caption 时返回对应图片 URL。

### 架构

```
图片 / 页面截图
  ↓
保存到静态文件服务（得到 image_url）
  ↓
VLM 生成 caption / OCR 提取文字
  ↓
caption 做文本 embedding
  ↓
caption + image_url + doc_id + page_no 一起写入 Milvus
  ↓
用 /search/dify 检索 caption，返回时带上 image_url
```

### 核心原则：图片 URL 在入库时生成

**重要**：图片 URL 不是检索时凭空得到的，而是在"图片入库阶段"就生成并存进去的。

```text
图片/页面截图
  → 保存到一个可通过 HTTP 访问的位置
  → 得到 image_url
  → VLM 生成 caption
  → caption 做文本 embedding
  → caption + image_url + doc_id + page_no 一起写入 Milvus
  → 检索 caption 时，通过 output_fields 把 image_url 一起返回
```

### 1. 图片 URL 从哪里来

先把图片保存到自己的文件服务。常见三种方式：

| 方式 | 说明 | 示例 |
|------|------|------|
| 方式一 | FastAPI / Flask / Nginx 静态目录 | `/kb-images/doc123/img0001.png` |
| 方式二 | MinIO / S3 对象存储 | `http://minio:9000/bucket/doc123/img0001.png` |
| 方式三 | 已有文档服务 | `/pdf_pages/doc123/page_0007.png` |

你们现有的服务：



### 2. 图片保存 + URL 生成代码

```python
import os
import shutil
from pathlib import Path


API_SERVER_BASE_URL = "http://10.1.120.36:5010"
IMAGE_ROOT = "/data/xinyang_kb/images"
IMAGE_URL_PREFIX = "/kb-images"


def make_image_id(doc_id: str, page_no: int, image_index: int) -> str:
    """生成图片唯一 ID"""
    raw = f"{doc_id}_p{page_no:04d}_img{image_index:04d}"
    return raw


def save_image_and_get_url(
    src_image_path: str,
    doc_id: str,
    page_no: int,
    image_index: int,
) -> dict:
    """
    保存图片并生成 URL

    Returns:
        {
            "image_id": "doc123_p0007_img0001",
            "image_path": "/data/xinyang_kb/images/doc123/doc123_p0007_img0001.png",
            "image_url": "/kb-images/doc123/doc123_p0007_img0001.png",
            "absolute_image_url": "http://10.1.120.36:5010/kb-images/doc123/doc123_p0007_img0001.png"
        }
    """
    image_id = make_image_id(doc_id, page_no, image_index)

    ext = Path(src_image_path).suffix.lower() or ".png"

    save_dir = Path(IMAGE_ROOT) / doc_id
    save_dir.mkdir(parents=True, exist_ok=True)

    filename = f"{image_id}{ext}"
    dst_path = save_dir / filename

    shutil.copyfile(src_image_path, dst_path)

    relative_url = f"{IMAGE_URL_PREFIX}/{doc_id}/{filename}"
    absolute_url = f"{API_SERVER_BASE_URL}{relative_url}"

    return {
        "image_id": image_id,
        "image_path": str(dst_path),
        "image_url": relative_url,
        "absolute_image_url": absolute_url,
    }
```

**注意**：Milvus 里建议存相对 URL，比如 `/kb-images/doc123/doc123_p0007_img0001.png`，返回时再统一转成绝对 URL。

### 3. 如果图片来源是 PDF，怎么抽出来

用 PyMuPDF 抽取 PDF 中的原始嵌入图片：

```python
import fitz  # PyMuPDF


def extract_pdf_images(
    pdf_path: str,
    output_root: str,
    doc_id: str,
) -> list[dict]:
    """
    从 PDF 中抽取所有图片

    Args:
        pdf_path: PDF 文件路径
        output_root: 图片输出根目录
        doc_id: 文档 ID

    Returns:
        [
            {
                "image_id": "doc123_p0007_img0001",
                "image_path": "/data/.../doc123/doc123_p0007_img0001.png",
                "image_url": "/kb-images/doc123/doc123_p0007_img0001.png",
                "page_no": 7,
                "image_index": 1
            },
            ...
        ]
    """
    doc = fitz.open(pdf_path)
    results = []

    for page_index in range(len(doc)):
        page_no = page_index + 1
        page = doc[page_index]

        image_list = page.get_images(full=True)

        for image_index, img in enumerate(image_list, start=1):
            xref = img[0]
            image_info = doc.extract_image(xref)

            image_bytes = image_info["image"]
            ext = image_info["ext"]

            save_dir = os.path.join(output_root, doc_id)
            os.makedirs(save_dir, exist_ok=True)

            image_id = f"{doc_id}_p{page_no:04d}_img{image_index:04d}"
            filename = f"{image_id}.{ext}"
            image_path = os.path.join(save_dir, filename)

            with open(image_path, "wb") as f:
                f.write(image_bytes)

            image_url = f"/kb-images/{doc_id}/{filename}"

            results.append({
                "image_id": image_id,
                "image_path": image_path,
                "image_url": image_url,
                "page_no": page_no,
                "image_index": image_index,
            })

    return results
```

**建议**：优先做**页面截图 URL**（方案二），再做原始图片抽取。因为 SOP、图纸、表格经常需要上下文，单独抽出来的图片可能丢掉标题、编号、说明文字。

### 6. 入库代码示例

```python
def insert_image_caption(
    collection,
    image_id: str,
    caption: str,
    embedding: list[float],
    image_url: str,
    doc_address: str,
    doc_name: str,
```

### 7. 检索时如何返回 image_url

关键是使用 Milvus 的 `output_fields`：

```python
def search_image_captions(
    collection,
    query_embedding: list[float],
    category: str | None = None,
    top_k: int = 10,
) -> list[dict]:
    """
    检索图片 captions，返回时带上 image_url
    """
    expr = None
    if category:
        expr = f'category == "{category}"'

    results = collection.search(
        data=[query_embedding],
        anns_field="embedding",
        param={
            "metric_type": "COSINE",
            "params": {"ef": 64},
        },
        limit=top_k,
        expr=expr,
        output_fields=[
            "content",
            "image_url",
            "doc_address",
            "doc_name",
            "doc_id",
            "page_no",
            "category",
            "content_type",
        ],
    )

    output = []
    for hit in results[0]:
        entity = hit.entity

        output.append({
            "score": float(hit.distance),
            "caption": entity.get("content"),
            "image_url": entity.get("image_url"),
            "doc_address": entity.get("doc_address"),
            "doc_name": entity.get("doc_name"),
            "doc_id": entity.get("doc_id"),
            "page_no": entity.get("page_no"),
            "category": entity.get("category"),
            "result_type": "image_caption",
        })

    return output
```

### 8. OpenWebUI Tool 改造

```python
def _convert_file_path_to_url(self, path: str) -> str:
    """将相对路径转换为完整 URL"""
    if not path:
        return path

    if path.startswith("http://") or path.startswith("https://"):
        return path

    if path.startswith("/pdf/"):
        return f"{self.API_SERVER_BASE_URL}{path}"

    if path.startswith("/kb-images/"):
        return f"{self.API_SERVER_BASE_URL}{path}"

    if path.startswith("/pdf_pages/"):
        return f"{self.API_SERVER_BASE_URL}{path}"

    return path


def _transform_search_result(self, data: dict) -> str:
    """转换搜索结果，增加 image_url"""
    new_data = []

    for item in data.get("results", []):
        new_item = dict(item)

        doc_address = new_item.get("doc_address", "")
        image_url = new_item.get("image_url", "")

        new_item.pop("source_file", None)

        # PDF URL
        new_item["url"] = self._convert_file_path_to_url(doc_address)

        # 图片 URL
        if image_url:
            full_image_url = self._convert_file_path_to_url(image_url)
            new_item["image_url"] = full_image_url
            new_item["markdown_image"] = f"![相关图片]({full_image_url})"

        new_data.append(new_item)

    return json.dumps(new_data, ensure_ascii=False)
```

## 方案三：ColQwen3 多向量视觉检索（推荐进阶）

**特点**：真正的视觉语义检索，按图像内容召回，不依赖文字。

### 架构

```
OpenWebUI Tool
  ↓
/search/multimodal
  ↓
tomoro-colqwen3-embed-4b 编码 query
  ↓
Milvus 2.5.14 搜 patch_vec
  ↓
按 page_id 聚合候选页面
  ↓
从本地/数据库取完整 patch embeddings
  ↓
应用层计算 MaxSim rerank
  ↓
返回 page_image_url / doc_address / page_no / score
```

### Collection 设计

**Milvus collection: `visual_page_patches`**（每 patch 一行）

| 字段 | 类型 | 说明 |
|------|------|------|
| patch_id | VARCHAR | 主键，如 `doc123_p0007_t0001` |
| page_id | VARCHAR | 页面 ID，如 `doc123_p0007` |
| doc_id | VARCHAR | 文档 ID |
| doc_name | VARCHAR | 文档名 |
| category | VARCHAR | 类别 |
| page_no | INT64 | 页码 |
| patch_index | INT64 | patch 序号 |
| patch_vec | FLOAT_VECTOR(320) | patch 向量 |
| page_image_url | VARCHAR | 页面截图 URL |
| doc_address | VARCHAR | 原 PDF 地址 |

**外部存储：完整页面 patch embeddings**

```
/data/visual_embeddings/doc123_p0007.npy
```

### 查询流程

```text
1. query → ColQwen3: [q_len, 320]
2. 每个 query token 去 Milvus 搜 top_n patches
3. 合并命中 page_id，得候选页面
4. 读候选页完整 patch embeddings
5. MaxSim: score(query, page) = sum_i max_j dot(q_i, p_j)
6. 排序返回 top_k 页面
```

### 优点

- 能按视觉内容、结构、版面、图表召回
- 检索质量最高
- 与旧文本检索隔离

### 缺点

- 工程复杂，需多向量检索 + 应用层 rerank
- 存储膨胀：一页可能几百到上千行
- 查询次数多
- 需要升级 Milvus 2.6 才能原生支持

### 适用场景

- 视觉结构搜索："左上角有蓝色框的流程图"、"包含三条曲线的实验图"
- 图纸、SOP、规格书中的配图检索

## 三种方案对比

| 对比项 | 方案一：文本+截图 | 方案二：OCR/Caption | 方案三：ColQwen3 |
|--------|-----------------|---------------------|------------------|
| **工程复杂度** | 低 ⭐ | 中 ⭐⭐ | 高 ⭐⭐⭐ |
| **上线速度** | 快 | 中 | 慢 |
| **本质** | 文本命中→返回截图 | 把图翻译成文字再搜 | 直接搜图片本身 |
| **检索对象** | 文本 chunk | 文字描述 caption | 视觉向量 |
| **信息损失** | 无 | 有，caption 压缩细节 | 无 |
| **入库成本** | 低，PDF渲染 | 中，VLM生成caption | 高，多向量编码 |
| **检索成本** | 低 | 低 | 高，MaxSim rerank |
| **存储成本** | 中（PNG截图） | 低（文本向量） | 高（多向量） |
| **需要升级 Milvus** | 否 | 否 | 是 |
| **需要 GPU** | 否 | 入库时需要 | 编解码都需要 |
| **适合场景** | 任何带文本的文档 | 简单图/流程图/照片 | 复杂图纸/扫描件/表格 |
| **推荐阶段** | 第一阶段快速上线 | 第二阶段补充 | 第三阶段增强 |

**一句话总结**：方案一最简单，方案二次之，方案三最难
