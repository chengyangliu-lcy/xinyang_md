### 1.2 嘉立创元器件 MCP 设计方案

#### 1.2.1 背景

在硬件开发、选型评估和故障分析过程中，工程师需要频繁查询元器件型号、规格参数、库存、价格、数据手册等信息。传统方式通常需要人工打开嘉立创/LCSC 网站进行检索，步骤较多，效率较低。

因此，计划设计一套面向 AI 助手的嘉立创元器件 MCP 工具，使 **芯阳webui平台、Claude Code、Codex、OpenCode** 等 AI 编程/研发助手能够直接接入嘉立创元器件数据源。用户只需通过自然语言提出查询需求，即可完成元器件搜索、详情查询和数据手册获取。

#### 1.2.2 元器件数据源摸底

目前已完成嘉立创/LCSC 相关公开接口的初步摸底，确认以下接口可用于 MCP 工具开发。

**接口 1：JLCPCB SMT 零件搜索接口**

  * 请求方式：POST
  * 接口地址：`jlcpcb.com/api/overseas-pcb-order/v1/shoppingCart/smtGood/selectSmtComponentList`
  * 输入参数：`keyword`、`pageSize`、`currentPage`
  * 输出内容：元器件列表，包括 C 编号、名称、型号、厂商、分类、库存、阶梯价格、数据手册链接、Basic/Extended 标识、RoHS 标识等。
  * 当前状态：可用，无需认证。


该接口适合作为元器件关键词搜索入口，例如搜索 “STM32F103”“0402 100nF 电容”“LM358”等。

**接口 2：LCSC 元器件详情接口**

  * 请求方式：GET
  * 接口地址：`wmsc.lcsc.com/ftps/wm/product/detail?productCode=C编号`
  * 输入参数：`productCode`，即 LCSC C 编号，例如 `C8734`
  * 输出内容：元器件名称、型号、厂商、分类、库存、最小包装数量、阶梯价格、数据手册直链、英文描述、完整技术参数列表、重量等。
  * 当前状态：可用，无需认证。


该接口适合根据具体 C 编号查询元器件完整信息，尤其适合获取参数表、价格、库存和数据手册链接。

**接口 3：数据手册获取接口**

数据手册获取能力复用 LCSC 元器件详情接口，从详情返回结果中提取 `pdfUrl` 字段，并将其封装为独立的 MCP 工具返回给用户。该设计可以降低接口复杂度，同时让用户通过自然语言直接获取元器件官方数据手册链接。

#### 1.2.3 MCP 工具设计方案

整体架构如下：
```text
用户自然语言请求
        ↓
AI 工具：芯阳webui平台 / Claude Code / Codex / OpenCode
        ↓
MCP Protocol（stdio）
        ↓
lcsc-mcp（Node.js / TypeScript）
        ↓
┌────────────────┬────────────────┬──────────────────┐
│ lcsc_search    │ lcsc_detail     │ lcsc_datasheet   │
│ 元器件搜索      │ 元器件详情查询    │ 数据手册获取       │
└────────────────┴────────────────┴──────────────────┘
        ↓
JLCPCB / LCSC 官方接口
```

计划实现 3 个 MCP 工具。

**工具 1：**` lcsc_search`**— 元器件搜索**

  * 功能：按关键词搜索嘉立创/LCSC 元器件库。
  * 输入参数：`keyword`、`page`、`page_size`
  * 后端接口：JLCPCB SMT 零件搜索接口。
  * 输出字段：C 编号、名称、型号、厂商、分类、库存数量、阶梯价格、Basic/Extended 标识、数据手册链接等。
  * 典型场景：


  * “帮我查一下 STM32F103C8T6”
  * “有没有 0402 100nF 电容”
  * “查一下 LM358 的可用型号和库存”


**工具 2：**` lcsc_detail`**— 元器件详情查询**

  * 功能：根据 C 编号查询元器件完整技术规格。
  * 输入参数：`product_code`，例如 `C8734`
  * 后端接口：LCSC 元器件详情接口。
  * 输出字段：名称、型号、厂商、分类、库存、最小起订量、阶梯价格、数据手册 PDF 直链、RoHS 状态、产品描述、完整技术参数列表等。
  * 典型场景：


  * “C8734 的详细参数是什么”
  * “这个芯片的工作电压和封装是什么”
  * “帮我看一下这个物料是否有库存”


**工具 3：**` lcsc_datasheet`**— 数据手册获取**

  * 功能：获取元器件官方数据手册 PDF 下载链接。
  * 输入参数：`product_code`
  * 后端接口：LCSC 元器件详情接口，提取其中的 `pdfUrl` 字段。
  * 输出字段：数据手册 PDF 直链 URL。
  * 典型场景：


  * “给我 STM32F103 的 datasheet”
  * “LM358 的手册在哪里下载”
  * “帮我获取 C8734 的数据手册”


#### 1.2.4 技术选型

方向| 选型| 原因  
---|---|---  
实现语言| TypeScript / Node.js| 生态成熟，便于通过 npm 全局安装和分发  
MCP SDK| `@modelcontextprotocol/sdk`| 官方 MCP SDK，适合快速实现标准 MCP Server  
传输方式| stdio| 芯阳webui平台、Claude Code、Codex、OpenCode 均可直接支持，无需额外启动 HTTP 服务  
部署方式| `npm install -g .`| 一次安装后可生成全局命令 `lcsc-mcp`，便于多工具复用  
工具形态| MCP Tool| 适合被 AI 助手通过自然语言自动调用  
  
#### 1.2.5 各工具接入方式

后续计划支持以下工具接入：

  1. **Claude Code**
```bash
claude mcp add --scope user lcsc -- lcsc-mcp
```

  2. **Codex CLI**
```bash
codex mcp add lcsc -- lcsc-mcp

  3. **OpenCode** 通过编辑配置文件接入：

```

## 二、风险与问题

### 2.1 搜索引擎覆盖范围有限

当前 SearXNG 部署后主要能够稳定检索国内可访问的网站。由于服务器网络环境限制，暂时无法稳定使用 Google Search、Bing、DuckDuckGo 等国外搜索源，因此对部分国外高质量技术资料、英文官方文档和海外论坛内容的覆盖能力有限。

后续如业务需要进一步提升搜索质量，可考虑为搜索服务配置代理或补充其他可访问的数据源，以增强对英文资料和国际站点的检索能力。

### 2.2 嘉立创/LCSC 接口存在限流风险

嘉立创/LCSC 相关接口目前可直接访问，但在高并发、大批量请求场景下，存在被限流或请求失败的可能。后续在 MCP 工具实现时，需要增加请求重试、超时控制、限速策略和错误提示，避免因短时间大量请求导致工具不可用。
