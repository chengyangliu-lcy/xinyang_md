#### 第一轮：疑问句 + 图片筛选

  * **脚本** : `filter_dataset.py`
  * **筛选条件** : 帖子内容必须包含**真正的疑问句** ，且包含**图片** 。


**疑问句识别逻辑** 涵盖多种问句类型，同时排除误匹配：

  * **是非问** : `吗/呢/吧` \+ 问号（例：“电感瞬间充电会产生高压吗？”）
  * **原因问** : `为什么/为何`（例：“为什么碳化硅MOSFET特别需要米勒钳位？”）
  * **方法问** : `怎么/如何/怎样`（例：“缓冲电路如何设计？”）
  * **具体信息** : `是什么/什么是/什么叫/什么意思/什么作用`
  * **选择/定位** : `哪些/哪个/哪里/何时`
  * **确认问** : `是否/是不是/能否`
  * **比较问** : `有何`（例：“有何区别？”）
  * **通用标记** : 中文问号后接有意义的中文。


**排除规则** : 排除“没什么”、“诸多”、“怎么也”、“怎么都”等非疑问模式的误匹配。

### 评分机制

### 📊 多模态问答评测机制总结

本次评测针对 **2025年 bb-world 帖子数据集** ，采用多角度混合评分机制。该机制旨在解决中文技术问答场景中“语义相近但表述不同”的评判难题，同时兼顾对专业术语、数值和单位的精确度要求。

#### 🎯 评测目标与结构

  * **输入** ：用户原始问题文本 + 图片路径列表。
  * **输出** ：模型生成的回答。
  * **标准** ：与数据集提供的参考答案进行比对。


#### 🧮 综合评分公式

最终得分采用加权求和方式计算，满分 1.0 分。

**公式：** $$final_score = 0.60 \times S_{semantic} + 0.25 \times S_{ROUGE-L} + 0.15 \times S_{Jaccard}$$

**权重设计逻辑：**

  * **语义相似度 (60%)** ：核心指标，判断回答的“意思”是否正确。
  * **ROUGE-L (25%)** ：辅助指标，判断回答是否覆盖了参考答案的主要表达片段。
  * **字符 Bigram Jaccard (15%)** ：约束指标，确保关键词、数值、单位等字面信息的准确性。


#### 🔍 三大核心评分维度

维度| 技术实现| 作用与优势| 示例场景  
---|---|---|---  
**语义相似度**|  基于 Embedding 模型计算余弦相似度。| **识别同义改写** 。解决字面不同但含义相同的问题。| 参考答案：“15kVrms是短时测试电压”  
模型回答：“15kV是安规耐压测试值”  
→ **高分**  
**ROUGE-L**|  基于最长公共子序列计算 F1 值。| **衡量内容覆盖度** 。反映模型回答是否包含参考答案的关键长句或短语结构。| 参考答案：“原边电压突变耦合到副边”  
模型回答：“原边电压突变通过电容传到副边”  
→ **中高分**  
**字符 Bigram**|  基于相邻字符对计算 Jaccard 相似度。| **强化术语精确性** 。不依赖分词，对中文技术术语、型号、数值单位敏感。| 检查是否遗漏“TL431”、“光耦”、“15kVrms”等具体词汇。  
→ **防止幻觉**  
  
Jaccard 相似度：中英文切割

#### 🤖 可选的 LLM 裁判

**LLM Judge** 评分项，但**默认不计入最终分数** ：

  * **功能** ：调用 LLM 对答案进行 0-1 的主观打分。
  * **用途** ：作为额外的参考字段保存，用于后续的人工校验或相关性分析。
  * **原因** ：LLM 评分成本高、复现性弱，因此主评分仍依赖确定性的数学指标。
