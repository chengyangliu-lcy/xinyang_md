# 联网搜索调研

## 背景与目标

当前 Agent 需要具备两类联网能力：一类是 **Web Search** ，用于根据问题检索候选网页；另一类是 **Web Fetch / Web Read** ，用于读取候选网页正文并交给 LLM 分析。现有实现主要依赖直接爬取搜索引擎 HTML 页面，以及用 `requests + BeautifulSoup` 读取网页正文。这套方案成本低，但稳定性较差，遇到搜索引擎反爬、JS 渲染页面、Cloudflare 校验页、403/429 限流等情况时容易失败。

## 一、当前实现的问题

### 1.1 Web Search 现状

当前有两套搜索机制：

机制| 实现| 问题  
---|---|---  
HTML 爬取 (`WebSearch`)| 直接爬 DuckDuckGo/Bing/Yahoo 的 HTML 页面| 遇到反爬会失败，结果不稳定  
Qwen 原生搜索 (`QwenSearchExecutor`)| DashScope API 的 `enable_search=True`| 需要付费，受阿里云配额限制  
  
HTML 爬取的 fallback 链是硬编码的：`duckduckgo → bing → yahoo`，三个都可能被反爬。

### 2.2 推荐：SearXNG 自部署

![image.png](https://cdn.nlark.com/yuque/0/2026/png/43223830/1778642070416-ddafa55e-a5d4-4c44-acf4-bd0d24ffb190.png)

![image.png](https://cdn.nlark.com/yuque/0/2026/png/43223830/1778642167108-fbaef7ef-f131-4300-8036-d91560ce4050.png)

**为什么是最佳选择：**

  * 完全免费开源（AGPL），Docker 一行命令部署
  * 聚合 70+ 搜索引擎（Google、Bing、DuckDuckGo、Wikipedia 等）
  * 天然分散反爬风险 — 不依赖单一引擎
  * 提供 JSON API，直接返回结构化结果
  * 无调用限制，无 API Key
  * LangChain、LlamaIndex 等框架有内置支持


**部署方式：**
```bash
docker run -d --name searxng -p 8888:8080 \
  -e SEARXNG_BASE_URL=http://localhost:8888 \
  searxng/searxng
```

**调用方式：**
```python
import requests

def search_searxng(query: str, limit: int = 10) -> list[dict]:
    resp = requests.get(
        "http://localhost:8888/search",
            params={"q": query, "format": "json", "categories": "general"},
            timeout=15
        )
        results = resp.json().get("results", [])
        return [
            {"title": r["title"], "url": r["url"], "snippet": r.get("content", "")}
            for r in results[:limit]
        ]
```

**注意事项：**

  * 需要一台服务器（VPS 或本地 Docker）
  * 上游搜索引擎仍可能限制高频请求，建议加缓存
  * 可通过配置文件调整引擎权重和超时

from duckduckgo_search import DDGS

**免费额度最高的官方 API：**

  * 1,000 次/月，每月重置
  * 注册即得 API Key：[https://api.search.brave.com](https://api.search.brave.com)
  * 返回结构化 JSON，支持 web/images/news 搜索

```python
import requests

def search_brave(query: str, api_key: str, limit: int = 10) -> list[dict]:
    resp = requests.get(
        "https://api.search.brave.com/res/v1/web/search",
        headers={"X-Subscription-Token": api_key},
        params={"q": query, "count": limit},
        timeout=15
    )
    results = resp.json().get("web", {}).get("results", [])
    return [
        {"title": r["title"], "url": r["url"], "snippet": r.get("description", "")}
        for r in results
    ]
```

## 三、免费 Web Fetch（网页抓取）方案

### 3.1 方案对比

方案| 是否免费| 是否支持 JS 渲染| 输出格式| 推荐定位  
---|---|---|---|---  
requests + BeautifulSoup| 免费| 否| text/html/text| 轻量快速首层

### 3.2 推荐：Crawl4AI

![image.png](https://cdn.nlark.com/yuque/0/2026/png/43223830/1778642782792-51d3cb88-f479-4915-b76d-98c63f5ec36f.png)

**完全开源的 LLM 专用爬虫：**
```bash
pip install crawl4ai
crawl4ai-setup  # 安装 Playwright 浏览器
```
```python
import asyncio
from crawl4ai import AsyncWebCrawler

async def fetch_crawl4ai(url: str) -> str:
    async with AsyncWebCrawler() as crawler:
        result = await crawler.arun(url=url)
        return result.markdown  # 干净的 Markdown 输出
```

**特点：**

  * Apache 2.0 开源，完全免费
  * Playwright 驱动，能处理 JS 渲染页面
  * 内置广告/侧边栏过滤
  * 支持代理配置、User-Agent 轮换
  * 内置缓存、会话管理
  * GitHub 38k+ stars，社区活跃


### 3.3 推荐：Jina Reader 作为远程兜底层 

**最简单的网页抓取方案：**

只需在目标 URL 前加 `https://r.jina.ai/`，即可获取干净的 Markdown 内容：
```python
import requests

def fetch_jina(url: str, max_chars: int = 8000) -> str:
    resp = requests.get(
        f"https://r.jina.ai/{url}",
        headers={"Accept": "text/plain"},
        timeout=20
    )
    return resp.text[:max_chars]
```

**特点：**

  * 无需 API Key（基础版免费）
  * 服务端处理 JS 渲染和反爬
  * 返回干净的 Markdown，直接给 LLM 消费
  * 支持 header 控制：`X-Return-Format`、`X-Wait-For-Selector` 等
  * 也有搜索服务：`https://s.jina.ai/{query}`


**限制：**

  * 免费版有速率限制
  * 不适合大批量抓取
