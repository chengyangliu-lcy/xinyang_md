# 4.3组会

# 异常元器件检测流程说明

## 1\. 核心流程

整体流程如下：

**数据准备 → 规则候选生成 → Candidate 阶段 → Verifier 阶段 → 决策融合 → 评测**

* * *

##  2\. 规则候选生成模块

### 2.1 核心思想

通过**多维分组** 与**统计规则** ，自动发现疑似异常元器件。

* * *

### 2.2 结构化特征构建

系统从多个视角对元器件进行分组，生成候选规则。

#### 分组视图

  * `prefix:family:designator_prefix`  
同前缀、同家族分组
  * `net:family:net_signature`  
同局部网络角色分组
  * `neighbor:family:neighbor_signature`  
同邻域拓扑分组
  * `desc_stem:family:description_stem`  
同描述词干分组
  * `symmetry:family:symmetry_signature`  
对称拓扑分组
  * `role:family:role_tag`  
同电路角色分组，例如：


  * `button_pullup`
  * `power_decoupling_cap`


* * *

### 2.3 规则打分

#### 主流值检测

组内主流值占比需满足：

  * `dominant_ratio >= 0.7`（默认值）


也就是：某个值在组内占比至少 70%，才可作为参考值。

* * *

### 2.4 `rule_score` 计算规则

#### 加分项

加分项| 分值| 说明  
---|---|---  
基础分| +0.25| 所有候选都有  
每多 1 个支持视图| +0.10| 最多加到 +0.30（即 3 个视图）  
比值 ≥ 2x| +0.15| 当前值明显偏离主流值  
对称视图支持| +0.10| 有 symmetry 视图支持  
角色 + 描述词干支持| +0.08| role 视图 + desc_stem 支持  
非标准 E 系列值| +0.04| 值不是标准系列值  
description 支持参考值| +0.10| description 也指向参考值  
Strong Rule| +0.10| 多视图 + 大组  
  
#### 减分项

减分项| 分值| 说明  
---|---|---  
description 支持当前 comment| -0.10| description 反而支持嫌疑值  
description 内部冲突| -0.15| description 中出现多个不同值  
Weak hint| -0.08| 候选仅为弱提示  
  
* * *

### 2.5 输出结果

输出为 `RuleHintSet`，包含：

  * `high_confidence_hints`
  * `weak_hints`


* * *

## 3\. 整体输入结构

输入给模型的内容由三部分组成：
```text
    ┌─────────────────────────────────────────────┐
    │ System Prompt（系统指令）                   │
    │ - 说明模型角色和任务                        │
    │ - 定义检测边界和证据规则                    │
    └─────────────────────────────────────────────┘
             +
    ┌─────────────────────────────────────────────┐
    │ User Prompt - Part 1: 元器件列表 + 网表     │
    │ - 所有元器件的 designator/comment/description │
    │ - connections_compact 网络连接信息          │
    └─────────────────────────────────────────────┘
             +
    ┌─────────────────────────────────────────────┐
    │ User Prompt - Part 2: 规则候选证据          │
    │ - 仅包含被规则筛为“可疑”的元器件            │
    └─────────────────────────────────────────────┘
```

## 6\. `CandidateEvidence` 数据结构
```python
class CandidateEvidence:
    candidate_designator: str        # 嫌疑元器件位号
    comment_value: str               # 当前 comment 值
    reference_value: str             # 参考值，例如组内多数电阻为 4.7K，则参考值为 "4.7K"
    supporting_groups: list[str]     # 支持异常判断的视图分组
    reference_designators: list[str] # 组内具有主流值的元器件位号
    description_support: str         # description 字段支持哪个值
    related_nets: list[str]          # 该元器件连接的网络列表
    rule_score: float                # 规则置信度分数，范围 [0, 1]
    strong_rule: bool                # 是否属于 strong rule
    max_group_size: int              # 最大组大小
    hint_level: str                  # "high" / "weak"
    support_view_count: int          # 支持视图数量
    role_tag: str                    # 电路角色标签
    symmetry_group_size: int         # 对称组大小
    e_series_flag: str               # E 系列标志
    desc_stem_support_count: int     # 描述词干支持计数
    max_ratio: float                 # 与参考值的最大比值
```

* * *

## 7\. 规则参数

参数| 默认值| 含义  
---|---|---  
`min_group_size`| 3| 组内成员少于 3 个则过滤  
`dominant_ratio`| 0.7| 主流值占比需 ≥ 70% 才能作为参考值  
  
* * *

## 8\. High / Weak Hint 的区别

维度| High Confidence Hints| Weak Hints  
---|---|---  
含义| 高置信度可疑元器件| 低置信度可疑元器件  
用途| 模型优先审查，采纳概率高| 供模型参考，需要更强证据才采纳  
数量限制| 无限制| 默认只取 Top 8（`rule_weak_topk = 8`）  
`rule_score`| 通常 ≥ 0.5| 通常 < 0.5  
  
* * *

## 9\. High Confidence 判定条件

满足以下任一条件，即可进入 **High Confidence** ：

条件| 说明| 示例  
---|---|---  
Strong Rule| `support_views >= 2` 且 `group_size >= 4` 且 `description != conflict`| R7 被 2 个视图标记，参考组里有 5 个 4.7K 电阻  
多视图 + 拓扑| `reference_count >= 2` 且（`views >= 2` 或存在拓扑信号）| C6 被 `net + symmetry` 支持，参考电容有 3 个  
大比值 + 多参考| `reference_count >= 3` 且 `ratio >= 2` 且 `description` 不冲突| R7 的 100K vs 4.7K，比值约 21x  
电源去耦电容 + 强证据| 是去耦电容，且 `description` 支持参考值，且（`strong_rule` 或 `views >= 2`）| C6 是去耦电容，description 也支持 100nF  
  
* * *

## 10\. Weak Confidence 的典型情况

出现以下情况时，通常归为 **Weak Hint** ：

  * `description_support == "supports_comment"`
  * `support_view_count == 1`
  * `reference_count < 2`
  * `ratio < 2`，且无其他强信号
  * 不满足任何 High Confidence 条件
